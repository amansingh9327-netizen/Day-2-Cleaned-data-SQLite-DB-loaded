## 🚀 Quick Start

### 1. Clone & setup

```bash
git clone https://github.com/YOUR_USERNAME/bluestock-mf-analytics.git
cd bluestock-mf-analytics

python -m venv venv
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Generate raw data

```bash
python generate_data.py
```

Generates 3 CSVs in `data/raw/` with intentional quality issues (duplicates, bad dates, mixed casing, anomalous values).

### 3. Clean the data

```bash
python clean_data.py
```

Outputs 3 cleaned CSVs to `data/processed/`. Console shows every cleaning action taken.

### 4. Load into SQLite

```bash
python load_db.py
```

Creates `bluestock_mf.db` with 6 tables (2 dimensions + 4 facts). Prints row-count verification automatically.

### 5. Run analytical queries

```bash
sqlite3 bluestock_mf.db < sql/queries.sql
```

Or open `bluestock_mf.db` in [DB Browser for SQLite](https://sqlitebrowser.org/) and paste from `sql/queries.sql`.

## 🧹 Data Cleaning Summary

### Task 1 — `nav_history.csv`

| Issue | Fix |
|---|---|
| Mixed / missing dates | Parsed `DD-MM-YYYY` → `datetime`; dropped unparseable rows |
| Duplicate (fund, date) rows | `drop_duplicates(subset=["amfi_code","date"])` |
| Negative / zero NAV | Set to `NaN`, then forward-filled |
| Holiday / weekend gaps | Built full weekday spine → forward-fill per fund |

### Task 2 — `investor_transactions.csv`

| Issue | Fix |
|---|---|
| 11 casing variants for transaction type | Mapped to canonical: `SIP / Lumpsum / Redemption / SWP / Dividend` |
| 4 mixed date formats | Flexible parser tried `DD-MM-YYYY`, `YYYY-MM-DD`, `DD/MM/YYYY`, `MM/DD/YYYY` |
| Negative amounts | Rows with `amount ≤ 0` dropped |
| Inconsistent KYC casing | Lowercased → validated against `{verified, pending, failed}` → else `unknown` |

### Task 3 — `scheme_performance.csv`

| Issue | Fix |
|---|---|
| Non-numeric returns (`N/A`) | `pd.to_numeric(errors='coerce')` → fill with category median |
| Return spikes (e.g. 999%) | Capped: `return_1y [-50,100]`, `return_3y [-30,80]`, `return_5y [-20,60]` |
| Expense ratio out of SEBI range | Capped to `[0.10, 2.50]`; `anomaly_flag = 1` set |

## 🗄️ Star Schema

```
                    ┌─────────────┐
                    │  dim_fund   │
                    │  (fund_id)  │
                    └──────┬──────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
    ┌─────▼──────┐  ┌──────▼──────┐  ┌─────▼──────┐
    │  fact_nav  │  │ fact_trans  │  │fact_perform│
    └─────┬──────┘  └──────┬──────┘  └─────┬──────┘
          │                │                │
          └────────────────┼────────────────┘
                           │
                    ┌──────▼──────┐     ┌────────────┐
                    │  dim_date   │     │  fact_aum  │
                    │  (date_id)  │     │            │
                    └─────────────┘     └────────────┘
```

| Table | Grain | Rows |
|---|---|---|
| `dim_fund` | 1 row per fund scheme | 10 |
| `dim_date` | 1 row per calendar day | 731 |
| `fact_nav` | 1 NAV per fund per business day | ~5,220 |
| `fact_transactions` | 1 row per investor transaction | ~4,999 |
| `fact_performance` | 1 row per fund per year | 30 |
| `fact_aum` | 1 AUM snapshot per fund per year | 30 |

## 📊 10 Analytical Queries

| # | Query | Business Purpose |
|---|---|---|
| Q1 | Top 5 Funds by AUM | Identify largest funds by assets managed |
| Q2 | Average NAV per Month | Track market-wide NAV trend MoM |
| Q3 | SIP YoY Growth | Compare SIP inflows 2023 vs 2024 |
| Q4 | Transactions by State | Geographic distribution of investment |
| Q5 | Funds with Expense Ratio < 1% | Surface low-cost fund options |
| Q6 | Best 3-Year CAGR | Rank funds by long-term returns |
| Q7 | Month-End NAV Trend | Portfolio reporting snapshots |
| Q8 | KYC Compliance Rate | Regulatory compliance check per fund |
| Q9 | Redemption Pressure by Quarter | Detect net outflow quarters |
| Q10 | Category Return Heatmap | Asset allocation decision support |
